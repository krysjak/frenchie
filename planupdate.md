# Фінальний план розвитку Frenchie (Супер-агент з пам'яттю та мультиагентною логікою)

Цей документ фіксує узгоджену архітектуру, кастомні функції та план реалізації нової версії автономного кодинг-асистента **Frenchie**. Мета — створити продукт, який є потужнішим за існуючі кодинг-агенти (такі як Hermes / Claude Code) завдяки унікальній системі пам'яті, динамічній багатоагентній взаємодії та рантайм-дебагу.

---

## 1. Архітектурні компоненти та кастомні фічі

### 🧠 1.1. Cognitive Memory System (CMS) — Чотирирівнева Пам'ять
Для швидкої орієнтації в проекті та збереження контексту впроваджується CMS:
1. **Карта Проекту (`FRENCHIE_MAP.json` / `PROJECT_MAP.md`)**:
   - Автоматично оновлюваний файл у форматі JSON/Markdown, що містить опис архітектури, ключові модулі, бази даних, роути та їхні зв'язки.
   - Агент зчитує карту при старті, щоб миттєво зрозуміти структуру проекту.
2. **Якорі Пам'яті (Contextual Memory Anchors)**:
   - Файл `.frenchie/memory_anchors.json` зберігає нотатки та "пастки", які агент створює для конкретних файлів коду (наприклад: *"При зміні `repl.py` переконайся, що UTF-8 сумісний з Windows"*).
   - При відкритті або редагуванні файлу пов'язаний якір пам'яті автоматично інжектується в системний промпт.
3. **Епізодична Пам'ять (Episodic Memory)**:
   - Зберігає хронологію дій поточної сесії (які файли змінювалися, які тести падали). Допомагає агенту виявляти першопричини багів (наприклад: *"Які мої останні зміни зламали цей імпорт?"*).
4. **Активний Блокнот (Working Memory Scratchpad)**:
   - Спеціальний тег `<scratchpad>` у промпті, куди агент записує проміжні спостереження, списки TODO та лог помилок під час виконання довгої та складної задачі.

---

### 🤖 1.2. Dynamic Multi-Agent Architecture (Багатоагентна система)
Замість однієї моделі, яка виконує все, система динамічно створює субагентів у рантаймі:
- **Frenchie-Orchestrator (Оркестратор)**: Керує процесом, розподіляє завдання субагентам, агрегує результати та спілкується з користувачем.
- **Frenchie-Planner (Планувальник / Архітектор)**: Досліджує кодову базу (AST, CMS, Grep), будує покроковий план у `task.md`. Не пише код.
- **Frenchie-Coder (Кодер)**: Отримує завдання від Planner та модифікує файли через інструменти `Write` / `Edit`.
- **Frenchie-Verifier (Критик / Тестувальник)**: Запускає тести (`pytest`, `npm test` тощо) та лінтери (`ruff`, `eslint`). Якщо виявляє баг — повертає задачу Coder'у з логами помилок для автоматичного виправлення (Self-Healing).

---

### 🔬 1.3. Dynamic Runtime Execution Probe (Динамічний Зонд Рантайму)
*Топ-фіча для дебагінгу:*
- Коли тестувальник бачить баг або Coder не розуміє поведінку функції, запускається інструмент `ProbeExecution`.
- Він робить тимчасову копію функції, обгортає її у трейсер на базі `sys.settrace`, запускає її з реальними/тестовими аргументами й формує **Runtime Snapshot**:
  - Значення локальних змінних на кожному кроці.
  - Типи вхідних та вихідних параметрів.
  - Реальний шлях виконання (яка гілка `if/else` чи цикл спрацював).
- snapshot передається моделі в контекст, що гарантує 100% точність виправлення логічних помилок.

---

### 📜 1.4. Git Design-Intent Context (Контекст Задуму Автора)
- Перед редагуванням будь-якої ділянки коду асистент автоматично аналізує `git blame` та повідомлення комітів (Commit messages) для цих рядків.
- Це дозволяє зрозуміти **чому** код був написаний саме так, та захищає від видалення важливих edge-case фіксів, які виглядають як "зайвий код".

---

### 🔄 1.5. Self-Development & Skills (Самонавчання)
- Директорія `.frenchie/skills/` містить успішно вивчені шаблони рішень ("Skill Recipes").
- Після вирішення складного багу Orchestrator проводить рефлексію і записує алгоритм вирішення. Наступного разу при схожому запиті навичка завантажується автоматично, економлячи час та токени.

---

### 💻 1.6. Codex-style Web UI Dashboard
Сучасний вебінтерфейс, що працює паралельно з CLI:
- **Ліва панель**: Чат з Orchestrator, налаштування та живий Mermaid-граф роботи субагентів (показує, який агент зараз думає чи працює).
- **Права панель (Split-screen)**: File Explorer, Code Editor з підсвіткою активного рядка та вкладка **Visual Diff** (інтерактивний git diff для перегляду та підтвердження пропонованих змін).
